# Claude Code CLI 核心指南：高效命令与快捷键简明简报

本报告旨在总结 Claude Code 命令行界面（CLI）的核心功能、操作逻辑及进阶技巧。Claude Code 不仅是一个对话式 AI 工具，更是一个能够深度集成到开发工作流中的自动化助手。通过掌握特定的斜杠命令（/）、前缀符号与快捷键，开发者可以显著提升代码编写、项目维护及上下文管理的效率。

---

## 执行摘要

Claude Code CLI 提供的功能远超简单的自然语言对话。其核心价值在于通过 **精确的文件引用**、**直接的 Bash 执行**、**结构化的规划模式** 以及 **智能的上下文压缩**，实现与本地开发环境的无缝协作。本简报详细介绍了如何通过 `/help` 和 `/status` 了解运行环境，利用 `@`、`!`、`#` 符号进行精准交互，以及通过 `/diff`、`/rewind` 和 `/go` 命令管理代码变更与自动化任务。

---

## 核心功能与命令深度分析

### 1. 基础探索与状态查看
在初次接触或排查问题时，了解工具的能力范围和当前运行状态至关重要。

| 命令 | 说明 | 应用场景 |
| :--- | :--- | :--- |
| **`/help`** | 调出命令选择器与完整帮助列表。支持左右方向键切换标签页（基础信息、默认命令、自定义命令）。 | 查找不熟悉的命令或探索新功能。 |
| **`/status`** | 查看版本、当前使用的模型、登录账号及详细的 Token 使用统计。 | 确认模型版本、排查连接问题或监控成本开销。 |

### 2. 精确交互前缀
为了让 Claude 精确理解开发者的意图，使用特定前缀可以减少自然语言的歧义并节省 Token。

*   **`@` (引用文件):** 明确指定处理对象（如 `@README.md`）。这使得 Claude 无需在项目中盲目搜索，回答更精准且经济。
*   **`!` (Bash 模式):** 以感叹号开头的输入会被视作 Shell 命令直接执行（如 `!pnpm run build`）。执行结果（包括成功、失败及日志）会自动进入对话上下文，避免手动复制粘贴。
*   **`#` (快速记忆):** 用于记录个人偏好、习惯或临时指令（如 `# 我的名字叫王大强`）。Claude 会创建[[记忆文件]]并建立索引，这些信息不随对话结束而丢失。

### 3. 权限管理与“规划模式”
Claude Code 支持通过 `Shift+Tab` 快捷键切换三种权限模式，平衡安全性与自动化。

*   **默认模式:** 修改文件或运行命令前均需逐一确认。
*   **规划模式 (Plan Mode):** **强烈建议养成的使用习惯。** 在该模式下，Claude 不会直接动手，而是先扫描项目并给出一份详尽的计划（涉及哪些文件、分几步走），用户确认后再执行。
*   **自动模式 (Auto Mode):** 高级自动化模式，授权 Claude 连续执行任务。

### 4. 代码管理与回退机制
针对代码变更的安全性，系统提供了类似 Git 的审查与回退功能。

*   **`/diff`:** 在提交代码前，通过[[差异视图]]查看所有已改动的内容。支持分文件查看详细差异，确保开发者对每一行变更了然于心。
*   **`/rewind`:** 会话与代码的“悔药”。可将代码仓库和对话记录同时回退到之前的某个快照点，并提供“仅回退代码”、“回退代码及会话”或“生成摘要”等选项。

### 5. 上下文优化与会话管理
长会话会导致 Token 爆掉或响应变慢，有效的管理至关重要。

*   **`/context`:** 视觉化展示当前[[上下文占用]]情况，包括系统提示词、工具、技能包、记忆文件及消息占用的比例。
*   **`/compact`:** 压缩算法。在保留任务主线的同时，总结并压缩前文，腾出上下文空间。
*   **`/clear`:** 清空当前对话开启“新局”，但历史对话仍存储在系统中。
*   **`/resume` 或 `claude -c`:** [[会话持久化]]。`claude -c` 用于从终端快速恢复最近一次会话，`/resume` 则提供列表供用户挑选特定历史记录。

### 6. 自动化任务执行
*   **`/go`:** [[目标驱动模式]]。设定一个完成条件（如“升级所有依赖并确保构建通过”），Claude 会自动开启循环：修改、测试、评估，直到满足条件为止。

*   **上下文：** 强调了对于进阶用户而言，单纯的自然语言对话仅利用了该工具的一小部分潜力，命令和快捷键才是提效的关键。

*   **上下文：** 解释为什么直接在对话框运行 Shell 命令更有利于维持上下文的整洁与准确性。

## 行动指南与最佳实践

1.  **首选规划模式：** 在处理大型重构或依赖升级任务时，务必先切换至 **Plan Mode**，审核计划后再授权执行，以规避潜在风险。
2.  **善用前缀符号：**
    *   使用 `@` 指定文件，降低搜索成本。
    *   使用 `!` 运行构建或测试命令，确保错误日志直接被 AI 捕获。
    *   使用 `#` 固化开发规矩或个人偏好。
3.  **定期清理与压缩：** 养成使用 `/context` 检查空间、使用 `/compact` 释放空间的习惯，防止在关键任务中出现上下文溢出。
4.  **审查变更：** 在完成任务后、正式提交代码前，强制执行一次 `/diff`，这不仅是代码审查，也是对 [[AI 工作质量]] 的最后把关。
5.  **会话持久化：** 充分利用 `claude -c` 和 `/resume`，将 Claude 当作一个长期的 [[协作项目成员]]，而非一次性的问答工具。
